The five things worth knowing before the lab

Quick theory first — each interactive module below builds directly on one of these. For the Pakistan-specific scam patterns, see the Learn guides and quiz.

01

Cybersecurity basics

Security rests on three goals — confidentiality, integrity, and availability. Most real-world breaches trace back to a human or configuration mistake, not an exotic technical exploit.

02

Social engineering

Phishing (email), smishing (SMS), and vishing (voice calls) all use the same trick: urgency and authority, not code, to get someone to hand over access.

03

Web security

The OWASP Top 10 catalogues the most common ways web apps fail — injection flaws and broken input handling near the top of the list.

04

IoT security

Smart cameras, plugs, and routers usually ship with weak factory defaults and rarely get updated once installed — which is exactly what makes them a soft target.

05

Cyber hygiene

Unique passwords, multi-factor authentication, and prompt updates stop the overwhelming majority of everyday attacks — no advanced tooling required.

See what an unvalidated input field lets through

This is a safe, fully simulated search box — nothing here touches a real database or a real browser. Try typing something ordinary, then try one of the payloads below to see how an unguarded input can be turned against an application.

01 What
The vulnerability
Unvalidated input lets attacker-supplied text change the meaning of a query or a page.
02 How
Exploited (conceptually)
A crafted string breaks out of the expected data and is treated as code or logic instead.
03 Demo
Below, and live in the lab
Try the payloads here, then the same class of bug against DVWA or Juice Shop in your VM.
04 Impact
Data exposure or hijack
Full-table reads via SQL injection, or session theft and page defacement via XSS.
05 Prevent
Validate at the boundary
Never trust input; validate type, length, and format before it reaches logic or the page.
06 Fix
For developers
Parameterised queries for SQL, output encoding/escaping for anything rendered as HTML.

    Verify a certificate
